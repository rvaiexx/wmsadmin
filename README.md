67Post Support System v2.0

Административное десктопное приложение для управления службой поддержки проекта 67Post. Система позволяет операторам взаимодействовать с курьерами (рабочими) в режиме реального времени, используя Firebase Firestore в качестве бэкенда и Firebase Cloud Messaging для отправки Push-уведомлений.
🚀 Основные возможности

Мониторинг активных чатов: Автоматическое сканирование базы данных на наличие новых обращений от пользователей.

Live-чат: Обмен текстовыми сообщениями в реальном времени с отображением ролей (Оператор, Рабочий, Система).

Push-уведомления: Интеграция с FCM для мгновенного оповещения курьеров об ответе поддержки.

Завершение сессий: Автоматизированный процесс закрытия тикета с отправкой системных сообщений и запросом оценки качества обслуживания.

Современный UI: Интерфейс построен на библиотеке customtkinter с поддержкой темной темы и адаптивной верстки.

🛠 Стек технологий

Язык: Python 3.x

GUI: customtkinter (обертка над Tkinter с современным дизайном)

Backend: Firebase Admin SDK (Firestore, Messaging)

Concurrency: Модуль threading для фонового обновления данных без блокировки интерфейса

📋 Требования и установка
1. Подготовка окружения

Убедитесь, что у вас установлены необходимые библиотеки:
Bash

pip install customtkinter firebase-admin

2. Настройка Firebase

Для работы приложения необходим файл ключа сервисного аккаунта Google Cloud:

Перейдите в консоль Firebase.

Скачайте JSON-файл с ключами (Service Account Key).

Переименуйте его в serviceAccountKey.json и положите в корневую папку с проектом.

3. Структура базы данных (Firestore)

Приложение ожидает следующую структуру в Firestore:

support_chats/{user_id}/messages/ — коллекция сообщений.

users/{email}/ — документы пользователей, содержащие поле fcmToken для отправки уведомлений.

🖥 Использование

Запустите приложение командой:
Bash

python admin_app.py

Слева появится список активных обращений (ID пользователей, у которых есть сообщения в базе).

При выборе чата подгрузится история переписки.

Для ответа введите текст в нижнее поле и нажмите "Enter" или кнопку "ПОСЛАТЬ".

Для закрытия тикета нажмите кнопку "Завершить сессию". Система автоматически отправит пользователю прощальное сообщение и форму обратной связи.

⚠️ Важные примечания

    Потоки: Обновление списка чатов происходит каждые 4 секунды, обновление активного диалога — каждые 2 секунды.

    Безопасность: Никогда не передавайте файл serviceAccountKey.json третьим лицам и не добавляйте его в публичные репозитории (используйте .gitignore).
